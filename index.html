01

User Problem

Fashion/styling apps recommend based on body type or trends ‚Äî they miss who the user is. Professional stylists understand personality, but cost $200+/session and aren't scalable. Users want personalized style advice that reflects their identity, not cookie-cutter outfit suggestions.

Competitive Landscape
Subscription Styling (Stitch Fix)

Human stylists + algorithms. Expensive ($20+ fee), slow turnaround, no personality depth.

AI Fashion Apps (Cher, Acloset)

Wardrobe management + outfit generation. No personality input, purely visual matching.

Virtual Try-On (Zara, ASOS)

AR-based clothing preview. Retailer-locked, no style guidance or personal direction.

ChatGPT-based Advice

Generic Q&A format. No structured personality extraction, no visual anchors.

Gap: No product connects personality understanding ‚Üí culturally resonant style archetype ‚Üí actionable recommendations

User Personas
Primary Gen-Z & Millennial Movie Lovers

Identify with characters; want style that expresses identity; active on social media; budget-conscious but willing to pay for personalization

Secondary Fashion-Curious Professionals

Want polished looks but lack styling knowledge; time-poor; value efficiency over browsing

02

Key Product Decisions

Conversation-First UX over Form-Based Input

Instead of surveys or quizzes, designed an AI "director" audition: an 8-12 round deep conversation that naturally extracts personality traits, aesthetic preferences, and lifestyle context. Users reveal more through dialogue than checkboxes.

Movie Characters as Style Anchors

Built a curated database of iconic characters with detailed style profiles (color palette, silhouette, key pieces). Characters serve as intuitive, culturally resonant references ‚Äî "You're a Margot Tenenbaum" is more actionable than "Try warm earth tones."

Multimodal Pipeline Architecture

Chained Gemini API calls: conversation analysis ‚Üí personality extraction ‚Üí character matching ‚Üí style report generation ‚Üí live video consultation. Each stage enriches context for the next.

User Problem

Candidates practice problems but lack feedback on: clarity of explanation, completeness of reasoning, and performance under pressure. Existing tools are either question banks with shallow AI extensions, or conversational tools that lack personalization depth.

02

Key Product Decisions

6-Agent Architecture over Monolithic LLM

Split the system into specialized agents ‚Äî each owns a domain (user context, role analysis, interview orchestration, scoring, progress tracking, resume generation). This enables modular iteration and better output quality vs. a single prompt.

Adaptive Follow-up System

4 follow-up types (Clarify / Probe / Challenge / Stress), each triggered by specific conditions. Follow-ups anchor on user's own words to avoid "AI feels robotic" problem. Probability randomized 40-70%.

3-Phase Pacing Model

Warm-up (0-25%) ‚Üí build confidence. Core (25-75%) ‚Üí core evaluation. Pressure (75-100%) ‚Üí test limits. If user scores ‚â•4/5 consecutively, skip warm-up; if ‚â§2/5, downshift to coaching mode.
